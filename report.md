# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 반도체 웨이퍼 식각 두께 맵 시각화 및 공간적 결함 패턴 진단기 (2차원 리스트와 함수 분리 활용)

이 프로그램을 만든 이유: 반도체 공정에서는 수많은 칩이 하나의 둥근 웨이퍼 위에서 만들어집니다. 이때 단순히 "불량이 몇 개 발생했다"를 아는 것을 넘어, "불량이 웨이퍼의 가운데에 몰려 있는지, 테두리에 몰려 있는지" 물리적인 위치(좌표)를 분석하면 장비의 어느 부품이 고장 났는지 원인을 찾을 수 있습니다. 이를 파이썬의 2차원 리스트와 반복문을 활용해 시뮬레이션해 보고 싶었습니다.


---

## 2. 문제 분해 및 모델링

### 입력
9x9 크기의 가상 웨이퍼 식각 두께 데이터 (2차원 리스트 형태)
사용자가 입력하는 '목표 두께'와 '허용 오차' (실수형 데이터)

### 처리
이중 for문으로 2차원 리스트를 스캔하며 각 좌표의 두께가 허용 오차를 벗어나는지 판정합니다.
불량으로 판정된 칩의 (X, Y) 좌표를 별도의 리스트에 수집합니다.
수집된 불량 칩들의 좌표와 웨이퍼 정중앙 좌표 사이의 거리를 피타고라스의 정리(** 0.5)로 계산하여 평균 거리를 구합니다.

### 출력
웨이퍼의 형태와 각 칩의 상태(정상, 과식각, 미식각, 빈공간)를 기호([ O ], [ - ], [ + ])로 시각화한 2D 맵
총 검사 개수, 정상 개수, 불량 개수 요약
평균 거리를 바탕으로 한 최종 결함 패턴 진단 결과 (가운데 불량 vs 테두리 불량)

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: wafer_grid

한 행의 의미: 실제 둥근 웨이퍼 상에서 특정 Y좌표(가로줄)에 일렬로 배열된 반도체 칩들의 두께 데이터 집합입니다.

각 열의 의미: 해당 가로줄에서 칩이 위치한 X좌표(순서)를 의미합니다.

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | X좌표 0번 위치의 칩 두께 | 0 (웨이퍼 바깥 빈 공간) |
| 1번 열 | X좌표 1번 위치의 칩 두께 | 96 (식각 두께 수치) |
| 2번 열 | X좌표 2번 위치의 칩 두께 | 100 (식각 두께 수치) |
| 3번 열 | X좌표 3번 위치의 칩 두께 | 101 (식각 두께 수치) |

이 2차원 리스트를 사용한 이유: 실제 둥근 웨이퍼의 물리적인 형태와 칩의 위치(X, Y 좌표)를 컴퓨터 상에 가장 직관적으로 구현하고 탐색할 수 있는 자료구조이기 때문입니다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| create_wafer_data() | 가상 웨이퍼 두께 데이터를 생성 | 없음 | 9x9 2차원 리스트 |
| print_and_evaluate_map() | 칩 합격/불량 판정 및 맵 시각화 출력 | 2차원 리스트, 목표두께, 오차	 | 총 개수, 정상 개수, 불량 좌표 리스트 |
| analyze_pattern() | 불량 좌표 거리를 계산해 패턴 진단	 | 불량 좌표 리스트, 웨이퍼 크기 | 진단 결과 텍스트 출력 |
| main() | 프로그램 전체 실행 흐름 제어	 | 없음 | 없음 |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: print_and_evaluate_map()

설명: 이 프로그램의 핵심 엔진입니다. 이중 for문을 통해 2차원 리스트의 모든 좌표를 탐색하고, if문을 사용해 칩의 상태를 판정합니다. 동시에 시각적인 맵을 화면에 그려주며, 다음 단계(패턴 분석)를 위해 불량 칩의 좌표만 따로 모아서 반환하는 등 '탐색, 판정, 시각화, 데이터 수집'의 핵심 역할을 모두 수행합니다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
if thickness == 0: 을 통해 칩이 없는 빈 공간은 연산에서 제외했습니다.
if abs(thickness - target) <= tolerance: 를 통해 두께가 오차 범위 내인지 확인하여 정상/불량을 판정했습니다.
if avg_distance < 2.5: 를 통해 불량 칩들의 평균 거리가 기준치보다 작으면 '가운데 불량', 크면 '테두리 불량'으로 진단했습니다.

### 반복문 사용 부분

무엇을 반복 처리했는가?
for y in range(size): 와 for x in range(size): 의 이중 반복문으로 2차원 공간의 모든 좌표를 탐색했습니다.
for x, y in defect_chips: 의 단일 반복문으로 수집된 불량 칩들의 좌표를 하나씩 꺼내어 거리를 계산했습니다.

### 조건문과 반복문이 함께 작동한 부분
이중 for문으로 2차원 리스트를 탐색하는 과정 내부에서, if문이 작동하여 현재 좌표의 데이터가 0인지, 정상인지, 불량인지를 매번 판단하고 그에 맞는 기호를 출력 문자열에 더해나가는 방식으로 함께 작동했습니다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력: 목표 두께 100, 허용 오차 3

예상 결과: 맵이 정상적으로 출력되고, 테두리와 중앙의 불량이 감지되어 패턴이 진단됨.

실제 결과: 예상대로 기호가 출력되고 '테두리 불량 패턴'으로 정확히 진단됨.


### 테스트 2: 예외 또는 경계 사례

입력: 목표 두께 100, 허용 오차 10 (오차를 매우 크게 줌)

예상 결과: 모든 칩이 오차 범위 안에 들어와 불량이 0개가 됨.

실제 결과: 맵 전체가 [ O ]로 출력되고, "모든 칩이 정상입니다!"라는 예외 처리 메시지가 정상 출력됨.


### 발견한 오류와 수정한 방법
초기 기획에서는 import math를 사용하여 수학 모듈을 불러오고, 클래스(Class)를 만들어 객체지향으로 구현하려 했습니다. 하지만 코드가 너무 복잡해지고 학교에서 배운 범위를 벗어나는 문제가 있었습니다. 이를 해결하기 위해 클래스를 완전히 삭제하고 순수 2차원 리스트와 함수(def)만으로 코드를 재구성했으며, 루트 계산도 모듈 없이 파이썬 기본 연산인 ** 0.5를 사용하도록 수정하여 직관성을 높였습니다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분: 웨이퍼의 둥근 형태를 띠면서도, 실제로 중앙과 테두리의 두께 차이가 명확하게 드러나는 9x9 크기의 가짜 데이터(2차원 리스트)를 생성해 달라고 요청하여 데이터를 확보했습니다.


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분: 초기에 Gemini는 Class를 활용한 객체지향 프로그래밍과 math 모듈 사용을 제안했습니다. 하지만 수행평가 기준(배운 문법 활용)에 맞추기 위해 이를 거절하고 코드를 다시 짜달라고 요청했습니다.


최종 코드에서 내가 직접 판단하거나 수정한 부분: 클래스 구조를 버리고, 2차원 리스트 안에 단순히 두께 '숫자'만 넣는 방식으로 데이터 구조를 내가 직접 단순화했습니다. 또한, 불량 칩의 좌표를 defect_chips 리스트에 튜플 형태로 저장하여 나중에 거리를 계산하는 로직은 내가 직접 설계하고 적용했습니다.

## 9. 한계와 개선 방안

현재 프로그램의 한계: 현재는 데이터가 코드 안에 고정(Hardcoding)되어 있어서 항상 같은 웨이퍼 맵만 분석할 수 있습니다. 또한 진단 패턴이 '가운데'와 '테두리' 두 가지로만 나뉘어 있어 세밀한 분석이 부족합니다.


다시 개선한다면 추가하고 싶은 기능: 파이썬의 random 모듈을 사용하여 실행할 때마다 두께 데이터가 무작위로 생성되는 기능을 추가하고 싶습니다. 또한, 특정 한쪽 모서리만 불량이 나는 '편심 불량' 패턴을 진단하는 로직도 추가해 보고 싶습니다.
