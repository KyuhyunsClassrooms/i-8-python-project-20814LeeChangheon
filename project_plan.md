# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 반도체 웨이퍼 식각 두께 맵 시각화 및 공간적 결함 패턴 진단기


---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.


내 설명: 반도체 식각 공정에서는 웨이퍼 표면을 균일하게 깎아내는 것이 수율의 핵심입니다. 하지만 장비 챔버 내부의 플라즈마 밀도 차이로 인해, 웨이퍼 중심부와 가장자리의 두께가 다르게 깎이는 현상이 빈번하게 발생합니다.
단순히 불량이 몇 개인지만 세는 것을 넘어, 2차원 리스트에 저장된 칩의 두께 데이터를 이중 반복문으로 스캔하여, 불량 칩들의 물리적 좌표를 분석해 장비 중심부 문제인지 가장자리 문제인지 결함의 공간적 패턴을 역추적하는 알고리즘을 구현하고자 합니다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 가상의 식각 두께 측정값이 들어 있는 2차원 리스트 (웨이퍼 형태에 맞춰 모서리는 빈 공간 처리)
- 목표 식각 두께 및 공정 허용 오차

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 탐색 및 판정: 이중 for문으로 2차원 리스트 공간을 X, Y 좌표로 스캔하며, 각 위치의 두께 수치가 허용 오차를 벗어나는지 if문으로 판정하여 불량 칩의 좌표를 별도의 리스트에 저장합니다.
- 공간 패턴 진단: 파이썬 기본 연산(제곱근 ** 0.5)을 사용하여, 불량 판정을 받은 칩의 좌표와 웨이퍼 정중앙 좌표 사이의 직선 거리를 계산합니다. 거리가 짧으면 중심부 불량, 멀면 가장자리 불량으로 분류하여 원인을 추론합니다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 정상, 과식각, 미식각, 빈 공간 등의 텍스트 기호로 콘솔에 정렬되어 출력되는 2D 웨이퍼 시각화 맵
- 총 검사 칩 개수, 정상 개수, 불량 개수 요약 출력
- 불량 칩의 위치 거리를 분석한 결함 패턴(가운데 불량 or 테두리 불량) 진단 결과

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: wafer_grid

```python
# 웨이퍼의 형태를 띄도록 0(빈 공간)과 두께 수치(숫자)가 담긴 2차원 리스트 구조
# wafer_grid = [
#     [0, 0, 96, 98, 99, 98, 96, 0, 0],
#     [0, 96, 100, 101, 100, 101, 96, 0, 0],
# ]
```

### 한 행의 의미
2차원 리스트의 한 행은 실제 둥근 웨이퍼 상에서 특정 Y좌표(가로줄)에 일렬로 배열된 반도체 칩들의 집합을 의미합니다.

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
|리스트의 행 인덱스 |웨이퍼 상의 Y 좌표  | 0, 1, 2 |
| 리스트의 열 인덱스 | 웨이퍼 상의 X 좌표 | 0, 1, 2 |
 | 각 칸의 데이터 | 해당 위치에 존재하는 칩의 두께 (숫자) | 0은 빈 공간, 숫자는 식각 두께 |

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| create_wafer_data() | 9x9 크기의 가상 웨이퍼 두께 데이터를 2차원 리스트로 생성 | 없음 | 2차원 리스트 데이터 |

| main() | 프로그램의 전체 실행 흐름 제어 | 없음 | 없음 |

## 6. 사용할 제어구조

### 조건문을 사용할 부분



내 계획:
1. if thickness == 0: 을 사용하여 웨이퍼 바깥의 빈 공간은 분석 및 출력 연산에서 제외시키는 방어 로직에 사용합니다.
2. if abs(thickness - target) <= tolerance: 를 통해 두께가 정상 허용치 이내인지 판정할 때 사용합니다.
3. 거리 계산 후, 거리가 기준치보다 작으면 중심부 불량, 크면 가장자리 불량으로 진단 로직을 분기할 때 사용합니다.

### 반복문을 사용할 부분



내 계획:
1. 물리적인 2차원 평면을 탐색해야 하므로, for문 안에 for문을 넣는 이중 반복문을 사용하여 전체 X, Y 좌표를 하나씩 훑으며 검사하는 알고리즘의 핵심 뼈대로 사용합니다.
2. 진단 결과를 출력할 때, 찾아낸 불량 칩들의 리스트를 순차적으로 보고서에 출력할 때 단일 for문을 사용합니다.
