# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 객체지향 기반 반도체 웨이퍼 식각 두께 맵 시각화 및 공간적 결함 패턴 진단기


---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.


내 설명: 반도체 식각 공정에서는 웨이퍼 표면을 균일하게 깎아내는 것이 수율의 핵심입니다. 하지만 장비 챔버 내부의 플라즈마 밀도 차이로 인해, 웨이퍼 중심부와 가장자리의 두께가 다르게 깎이는 현상이 빈번하게 발생합니다.
단순히 불량이 몇 개인지만 세는 것을 넘어, 각 반도체 칩을 객체로 모델링하고 2차원 리스트 공간 안에서 이중 반복문으로 스캔하여, 불량 칩들의 물리적 좌표를 분석해 장비 중심부 문제인지 가장자리 문제인지 결함의 공간적 패턴을 역추적하는 알고리즘을 구현하고자 합니다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 가상의 식각 두께 측정값이 들어 있는 2차원 리스트 (웨이퍼 형태에 맞춰 모서리는 빈 공간 처리)
- 목표 식각 두께 및 공정 허용 오차

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 물리적 모델링: 단순 숫자였던 2차원 리스트의 데이터들을 칩 클래스 객체로 변환하여 새로운 2차원 리스트에 배치합니다.
- 탐색 및 판정: 이중 for문으로 2차원 리스트 공간을 X, Y 좌표로 스캔하며, 각 칩 객체의 두께가 허용 오차를 벗어나는지 if문으로 판정하여 상태를 업데이트합니다.
- 공간 패턴 진단: 파이썬 내장 math 모듈을 사용하여, 불량 판정을 받은 칩의 좌표와 웨이퍼 정중앙 좌표 사이의 직선 거리를 계산합니다. 거리가 짧으면 중심부 불량, 멀면 가장자리 불량으로 분류하여 원인을 추론합니다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 정상, 과식각, 미식각, 빈 공간 등의 텍스트 기호로 콘솔에 정렬되어 출력되는 2D 웨이퍼 시각화 맵
- 총 검사 칩 개수, 정상 개수, 불량 개수 요약 출력
- 불량 칩의 위치 거리를 분석한 결함 패턴(가운데 불량 or 테두리 불량) 진단 결과

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: wafer_grid

```python
# 초기에는 단순 두께 숫자 데이터였으나, 클래스를 통해 아래와 같이 객체들이 담긴 2차원 배열로 변환됨
# wafer_grid = [
#     [None, None, Chip(x=2,y=0,thick=105), Chip(x=3,y=0,thick=106), ...],
#     [None, Chip(x=1,y=1,thick=102), Chip(x=2,y=1,thick=101), ...],
# ]
```

### 한 행의 의미
2차원 리스트의 한 행은 실제 둥근 웨이퍼 상에서 특정 Y좌표(가로줄)에 일렬로 배열된 반도체 칩들의 집합을 의미합니다.

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
|리스트의 행 인덱스 |웨이퍼 상의 Y 좌표  | 0, 1, 2 |
| 리스트의 열 인덱스 | 웨이퍼 상의 X 좌표 | 0, 1, 2 |
| 각 칸의 데이터 | 해당 위치에 존재하는 칩 객체	 | 객체 내부 속성으로 두께, 불량여부 포함 |

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
|class Chip  | 개별 칩의 좌표, 두께, 상태를 저장하는 데이터 클래스 | x좌표, y좌표, 두께 | 칩 객체 |
| class WaferAnalyzer | 웨이퍼 전체를 관리하고 분석하는 메인 클래스 | 2차원 두께 데이터 리스트 | 분석기 객체 |
| evaluate_chips() | 이중 for문으로 모든 칩을 순회하며 합격 및 불량 판정 | 목표 두께, 오차 허용치 | 내부 객체 상태 업데이트 |
|analyze_pattern()|불량 칩들의 중심점 거리를 계산해 공간적 결함 원인 분석|없음|불량 패턴 이름 및 진단 결과 텍스트|
|print_map()|좌표에 맞춰 칩의 상태를 텍스트 기호로 정렬하여 출력|없음|터미널 2D 맵 출력|

## 6. 사용할 제어구조

### 조건문을 사용할 부분



내 계획:
1. if chip is None: 을 사용하여 웨이퍼 바깥의 빈 공간은 분석 및 출력 연산에서 제외시키는 방어 로직에 사용합니다.
2. if abs(chip.thickness - target) <= tolerance: 를 통해 두께가 정상 허용치 이내인지 판정할 때 사용합니다.
3. 거리 계산 후, 거리가 기준치보다 작으면 중심부 불량, 크면 가장자리 불량으로 진단 로직을 분기할 때 사용합니다.

### 반복문을 사용할 부분



내 계획:
1. 물리적인 2차원 평면을 탐색해야 하므로, for문 안에 for문을 넣는 이중 반복문을 사용하여 전체 X, Y 좌표를 하나씩 훑으며 검사하는 알고리즘의 핵심 뼈대로 사용합니다.
2. 진단 결과를 출력할 때, 찾아낸 불량 칩들의 리스트를 순차적으로 보고서에 출력할 때 단일 for문을 사용합니다.
